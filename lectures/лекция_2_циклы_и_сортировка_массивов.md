# Лекция 2: Циклы и сортировка массивов

## 1. Что такое цикл

Цикл — это способ выполнить один и тот же набор действий несколько раз.

Проще говоря: если вам нужно повторить одно и то же действие много раз, вместо копирования кода вы используете цикл.

### Пример из жизни

Представьте, что вам нужно 5 раз сказать «Привет»:

Без цикла:
- Привет
- Привет
- Привет
- Привет
- Привет

С циклом:
- «Повтори 5 раз: скажи Привет»

Цикл делает именно это — повторяет действия.

---

## 2. Как работает цикл

Цикл обычно состоит из:

1. **Начального значения** — с чего начинаем
2. **Условия** — когда остановиться
3. **Шага** — как изменяется значение

### Псевдокод цикла

```
для i от 0 до 4:
    вывести "Привет"
```

### Объяснение словами

- Сначала `i = 0`
- Проверяем: `i <= 4` — да
- Выполняем действие
- Увеличиваем `i` на 1
- Повторяем, пока условие выполняется

---

## 3. Массив и зачем его сортировать

**Массив** — это список значений.

Пример:

```
[5, 2, 4, 1]
```

**Сортировка** — это упорядочивание элементов.

Например:

```
[1, 2, 4, 5]
```

---

## 4. Одна итерация сортировки

Давайте посмотрим, что происходит за **один проход по массиву**.

### Идея

Мы сравниваем соседние элементы и меняем их местами, если они стоят неправильно.

### Псевдокод одной итерации

```
массив = [5, 2, 4, 1]

для i от 0 до длина(массив) - 2:
    если массив[i] > массив[i + 1]:
        поменять местами массив[i] и массив[i + 1]
```

### Объяснение словами

Берём массив:

```
[5, 2, 4, 1]
```

1. Сравниваем 5 и 2 → 5 больше → меняем → `[2, 5, 4, 1]`
2. Сравниваем 5 и 4 → 5 больше → меняем → `[2, 4, 5, 1]`
3. Сравниваем 5 и 1 → 5 больше → меняем → `[2, 4, 1, 5]`

После одной итерации получаем:

```
[2, 4, 1, 5]
```

👉 Самый большой элемент «всплыл» в конец.

---

## 5. Полная сортировка двойным циклом

Одной итерации недостаточно, потому что массив всё ещё не отсортирован.

Нужно повторить процесс несколько раз.

### Псевдокод полной сортировки

```
массив = [5, 2, 4, 1]

для j от 0 до длина(массив) - 1:
    для i от 0 до длина(массив) - 2:
        если массив[i] > массив[i + 1]:
            поменять местами массив[i] и массив[i + 1]
```

---

## 6. Объяснение двойного цикла словами

### Внешний цикл

Говорит: «Повтори проход по массиву несколько раз».

### Внутренний цикл

На каждом проходе сравнивает соседние элементы и при необходимости меняет их местами.

### Как это выглядит по шагам

Исходный массив:

```
[5, 2, 4, 1]
```

#### Первый проход

Получаем:

```
[2, 4, 1, 5]
```

#### Второй проход

Получаем:

```
[2, 1, 4, 5]
```

#### Третий проход

Получаем:

```
[1, 2, 4, 5]
```

Теперь массив отсортирован.

## 7. Подсчёт сложности алгоритма

Когда мы считаем сложность алгоритма, мы оцениваем, сколько действий он выполняет в зависимости от размера входных данных.

Размер массива обычно обозначают буквой **n**.

Например:
- если в массиве 4 элемента, то `n = 4`
- если в массиве 100 элементов, то `n = 100`

### Зачем считать сложность

Это помогает понять:
- насколько быстро работает алгоритм
- как алгоритм ведёт себя на больших массивах
- какой алгоритм лучше использовать

---

### Сложность одной итерации

В одной итерации мы один раз проходим по массиву и сравниваем соседние элементы.

Псевдокод:

```
для i от 0 до длина(массив) - 2:
    если массив[i] > массив[i + 1]:
        поменять местами массив[i] и массив[i + 1]
```

Если в массиве `n` элементов, то выполняется примерно `n - 1` сравнений.

Такую сложность записывают так:

```
O(n)
```

### Объяснение простыми словами

- Если элементов стало в 2 раза больше, то и работы примерно в 2 раза больше
- Это значит, что алгоритм растёт **линейно**

---

### Сложность полной сортировки

При полной сортировке работает двойной цикл:
- внешний цикл повторяет проходы
- внутренний цикл делает сравнения внутри каждого прохода

Псевдокод:

```
для j от 0 до длина(массив) - 1:
    для i от 0 до длина(массив) - 2:
        если массив[i] > массив[i + 1]:
            поменять местами массив[i] и массив[i + 1]
```

Что происходит:
- внешний цикл выполняется примерно `n` раз
- внутренний цикл тоже выполняется примерно `n` раз
- всего получается примерно `n * n` действий

Такую сложность записывают так:

```
O(n^2)
```

### Объяснение простыми словами

- Если элементов стало в 2 раза больше, работы становится примерно в 4 раза больше
- Если элементов стало в 10 раз больше, работы становится примерно в 100 раз больше

Поэтому двойной цикл может быть медленным на больших массивах.

---

### Небольшой пример

Пусть есть массив из 5 элементов.

- Одна итерация: около 4 сравнений
- Полная сортировка: по порядку роста около 25 действий

Пусть есть массив из 100 элементов.

- Одна итерация: около 100 действий
- Полная сортировка: около 10 000 действий

👉 Поэтому важно понимать не только **что делает алгоритм**, но и **сколько операций он выполняет**.

---

## 8. Простое объяснение идеи сортировки

Можно представить, что:

- мы снова и снова проходим по списку
- каждый раз самый большой элемент сдвигается вправо
- через несколько проходов все элементы оказываются на своих местах

---

## 9. Итог

- Цикл позволяет повторять действия
- Один цикл — это один проход
- Два цикла — это повторение проходов
- Сортировка — это многократное сравнение и обмен элементов
- Одна итерация имеет сложность `O(n)`
- Полная пузырьковая сортировка имеет сложность `O(n^2)`
